vlan技术

VLAN的特点：分割广播域，划分用户组，

  1. 端口的类型
    1. access；该端口只能属于一个vlan，发送的帧不带vlan标签，一般用于链接PC使用
    2. trunk；允许多个VLAN通过，发送的帧一般是带有VLAN标签的，一般用于交换机之间的互联端口使用
    3. hybrid；兼具access和trunk端口的功能和可以允许多个VLAN的报文发送时不打标签
  2. PVLAN技术
    1. Private VLAN；属于主VLAN 中的端口，可以与任意端口通讯
    2. Isolated VLAN；彼此之间不能通信，只能与private vlan口通讯
    3. Community VLAN；该端口下可以互相通信，可以与private vlan口通讯，不能跟 Isolated VLAN口通信
  3. Super VLAN
    1. Super VLAN；只能包含Sub VLAN，不能做为其他Super VLAN的Sub VLAN使用，不能把vlan 1作为本端口，基于本端口配置的ACL和QOS配置不对Sub VLAN生效
    2. Sub VLAN；包含实际的物理端口，不能配置路由接口，不能配置IP地址
  4. Native VLAN
    5. 属于Native VLAN的数据帧在Trunk链路上不携带VLAN标签传输，不带标签的数据经过Trunk链路会打赏Native VLAN所属的Vlan ID TAG形成数据帧

NAT技术

NAT的概念：是把内网私有网络转换成合法的公网IP地址

  1. NAT的用途
    1. 解决地址空间不足的问题
    2. 私有IP地址网络与公网网络互联
    3. 使用未注册的公网地址与公网互联
    4. 网络改造中，避免更改地址带来的风险
  2. NAT的分类
    1. 静态NAT：手动建立一个内网IP地址到公网IP地址的映射关系――常常用于内网一个设备需要被公网访问
    2. 动态NAT：将一个内网IP地址转换成一组公网IP地址(地址池)中的一个IP地址――常常用于整个公司共用多个公网IP地址
    3. 超载NAT：动态NAT的一种特殊的形式，利用不同端口号将多个内网多个IP地址转换为一个公网IP地址，也叫PAT,NAPT或端口复用NAT
    4. 端口映射NAT：常用于内网网络中的服务器，将某个服务端口映射到公网地址上，一个公网IP可以映射多个内网服务

二层交换机工作原理

早期的交换机使用的是集线器，无法分割冲突域。现代使用的是交换机，可以分割冲突域。以太交换机是需要MAC地址表进行数据帧的转发的

  1. 二层交换机根据MAC地址将帧选择性的从接收端口转发到目的端口
    1. 学习记录：记录原MAC地址和接收端口的对应关系，构建MAC地址表
    2. 查表转发：交换机收到一个数据帧后，读取该数据帧的目的MAC，并查看MAC地址表(有三种转发行为)
      • 2.1 转发：MAC地址表中有该MAC地址的记录，按照其对应的端口进行转发
      • 2.2 泛洪：如果没有，将从某个接口收到的数据流向除该接口之外的所有接口发送出去
      • 2.3 丢弃：如果数据帧的源MAC地址和目的MAC地址相等则丢弃
  2. MAC地址表的维护和老化
    1. 当交换机的接口DOWN掉后，与该接口相关的MAC地址都会被清楚
    2. 当PC一段时间内（默认是5分钟）没有发送任何报文，交换机会将该PC的MAC地址表项自动删除

路由的原理

路由：在网络上告诉你路怎么走
路由的下一跳：路由设备进行转发，下一台设备的端口IP地址；路由表决定下一跳地址，PC客户端的默认下一跳通常称为“网关”；下一跳地址是路由设备根据路由表选择的最佳路径

  1. 路由表：路由表的构成为静态路由，动态路由和直连路由
    1. 静态路由
      • 1.1 需管理员手动配置的固定路由
      • 1.2 无法对网络拓扑改变而变化，需人为调整所以适用于小型简单的网络
      • 1.3 当和动态路由发生冲突时，以静态路由为准
    2. 动态路由
      • 2.1 路由设备之间使用某种动态路由协议，利用收到的路由信息生成路由表
      • 2.2 在网络拓扑改变时，能够自动调整路由信息
      • 2.3 会在网络中产生额外的流量，给路由设备带来额外的cpu和内存开销
    3. 直连路由
      • 3.1 通过活动接口配置IP形成的直连路由
      • 3.2 直连路由表示路由设备所直接链接的网络
  2. 度量值(Metric)
    1. 路由协议用来衡量路径优劣的参数，表示到达目的地的代价总和
    2. 影响度量值的因素：线路带宽、跳数、线路延迟、线路使用率、线路可信度
  3. 管理距离（Administrative Distance）
    1. 也叫优先级，正常情况下管理距离越小，它的优先级就越高，也就是可信度越高
    2. 该值越低，优先级或可信度越高，只有该值越低的路由会被添加到路由表
    3. 该值是一个0~255的整数，0是最可信赖的，而255是路由不可达，该值是由厂商自定义的
  4. 路由表生成的原则
    1. 首先比较目标网络，不同添加路由表，如果相等进行下一步
    2. 其次比较管理距离，不同添加路由表，如果相等进行下一步
    3. 然后比较度量值，不同添加路由表，如果相等进行下一步
    4. 最后负载均衡，均添加路由表，路径之间等价分担流量
